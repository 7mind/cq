## Review round 1 — finding (GOAL-TITLE-01, minor)

The Claude producer advertises a SEPARATE looser SDK-tool input schema
(`submitPlanSchema`) than the strict `ProducerOutputSchema` it re-validates
against. The SDK strips the model's tool input to the advertised schema BEFORE
the handler runs, so omitting `title` from `submitPlanSchema` silently dropped
the title → strict re-validation failed → handler returned isError → model
retried until the 40s dispatch timeout → outcome `errored`. Surfaced as two
REAL-SDK MockAnthropic integration tests timing out at 40s. Fixed by adding
`title` to `submitPlanSchema`; both tests then pass in ~1.6s. The continuation
phase subagent advertises `{payload:z.unknown()}` (no stripping) and the Codex
relay forwards an opaque payload validated against `ProducerOutputSchema`
directly — neither needed the analogous fix. No second round required.

Also caught during fixture work: `canonical-ledgers.test.ts` builds a goal via a
generic per-ledger `create` map (line 127) and embeds a raw on-disk goals.md
fixture (line 324); both needed `title` or the now-required field rejects the
goal at the store boundary / at parse.

## Divergence-guard implication (accepted, not weakened)

Changing `GOALS_SCHEMA` means an existing dev `docs/goals.md` written under the
OLD schema (no `title`) now fails the bootstrap schema-divergence check
(`BootstrapViolationError`) and the server refuses to start. `docs/goals.md` /
`docs/ledgers.yaml` are gitignored regenerable artifacts; a developer holding a
stale file must `rm docs/goals.md docs/ledgers.yaml` once. No such file existed
in this worktree (confirmed absent + gitignored). Tests use fresh tmp cwds, so
they are unaffected.
